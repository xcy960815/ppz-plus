# UI 迁移约束

更新时间：2026-06-30

本文档记录从旧 PPZ 迁移页面类 UI 时必须遵守的表现层规则。

## 基本原则

- 只要是页面类 UI，包括 Webview 页面、表详情页、SQL 终端页、导入导出页面等，都必须使用 `ppz-plus` 自己的新架构代码实现。
- 不允许直接复用旧 PPZ 页面运行时代码、Vue 组件、Webview 脚本或样式文件。
- 允许为遵守 ppz-plus 新分层架构重写实现方式，但最终表现要贴近旧 PPZ，而不是仅满足功能可用。

## 视觉和交互复刻

- 页面类 UI 的功能和样式必须以旧 PPZ 源码为准做完整复刻。
- 结构、布局密度、颜色、间距、图标语义、按钮文案、交互流程和状态提示都要尽量对齐旧 PPZ。
- 迁移旧 PPZ 已有界面能力时，视觉样式、布局密度、字段排列、按钮文案和交互结构必须优先参考旧项目当前代码或 `v0.5.1-beta` 对应实现。
- 目标是“按旧项目样式一模一样重建体验”，不是重新设计一套新的 ppz-plus 风格。

## 数据库能力占位

- 如果旧界面包含多数据库选项，而 ppz-plus 当前阶段只支持 MySQL，可以保留旧界面的视觉占位。
- 不允许提前打开未完成的数据库能力。
- 数据库是否可用必须服从当前 capability 声明和实际实现状态。
